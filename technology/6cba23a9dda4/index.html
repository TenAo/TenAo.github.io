【博客搭建记录】使用Typora与PicGo打出Combo

2021-01-07 技术 Views
文章目录

本文最后更新于 <span id="expire-date"></span> 天前，文中部分描述可能已经过时。

  1. 打开Typora偏好设置>图像。

    因为我经常图方便，调用Windows自带的截图功能，直接粘贴进Typora，不存本地图片，所以这里插入图片时才选择复制到指定路径，没有相关需求的也可以选择上传图片，这样在插入时图片就会自动上传。

    上传服务图方便就选择app了。

  2. 安装完成，根据提示进行设置。需要去Github生成token。

    点击确定保存设置，并设为默认图床。然后在Typora的图像设置里验证图片上传选项。验证成功会在repo里上传两张typora的图片。

  3. 验证成功。进入博文，点击上传所有本地图片。

    提示上传成功。

使用了一下觉得开App还是很麻烦，决定还是用PicGo-Core(command line)。而且感觉github传图片经常出奇奇怪怪的问题，最常见的比如图片明明传上去了但是Typora里链接并没有自动更换。决定换Gitee试一试。

参考教程：https://www.jianshu.com/p/d7bf3f9da6db

简单步骤。

  1. Typora内选择PicGo-Core(command line)，点击下载。

  2. 将程序加入系统环境变量。位置在C:\Users\用户名\AppData\Roaming\Typora\picgo\win64

  3. 在cmd内根据提示设置，或者直接Typora内打开config文件设置。（推荐）

    {
  "picBed": {
    "current": "gitee",
    "gitee": {
      "repo": "username/仓库名",
      "branch": "master",
      "token": "",
      "path": "img",
      "customPath": "yearMonth",
      "customUrl": ""
    },
    "uploader": "gitee",
    "transformer": "path"
  },
  "picgoPlugins": {
    "picgo-plugin-gitee-uploader": true
  },
}

  4. 验证上传，成功。以及虽然我传图片一般都几十k，但是以防万一还是去gitee开了gitee pages，万一哪天要传我家仓鼠的萌萌Gif呢你说是吧。
