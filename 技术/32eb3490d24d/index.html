2021.01.04

已经更换了更适合自己的Cards主题，此内容不再更新。

Gridea虽然很方便，但还是不太喜欢用客户端，换回Hexo。

版本信息

Hexo：5.3.0

Next：7.8.0

Scheme： Gemini

  1. 设置网站语言。Next主题提供的简体中文的参数应该为zh_CN，即我们需要在站点config文件内设置

    languages: zh_CN

  2. 使用Hexo+Typora进行图片插入。参考博文：typora + hexo博客中插入图片

  3. 对博文的链接进行优化。参考博文：Hexo博客文章链接优化

    优化前长这样：

    这个日期跟文章名称，好家伙，血压拉满。

    处理如下：

    npm install hexo-abbrlink --save

    在站点Config内设置：

    会在generate的时候自动在文章内生成一个abbrlink参数值，也可以自己修改这个值，比如本篇博文我就设置为了hexo_optimize。

    试了一下，本地可以跳转链接，同步到Github Pages后就不可以了，懒得去研究，最后还是使用自带的permalink来做，具体参数见Hexo官方文档，在站点Config文件内修改设置如下。

    :hash后一定要加/，否则点击链接可能会下载文件，Hexo官方文档里评论区里找到的解释是：不带后缀名会导致有的web server发送错误的MIME类型，浏览器接收到非HTML的MIME，会默认进行下载。

    PS：如果已经安装了abbrlink，记得npm uninstall hexo-abbrlink，不然以后hexo g还是会自动在Front matter生成随机值。

  4. 版权协议样式优化。文件位置：

    ..\themes\next\source\css\_common\components\post\post-copyright.styl

    版权协议内容优化。文件位置：

    ..\themes\next\layout\_partials\post\post-copyright.swig

  5. 修改内容区域宽度。参考博文

  6. 链接颜色设置，标题样式优化等。在以下目录（选择自己的scheme，我的是Gemini）末尾添加css代码，覆写layout。

    ..\themes\next\source\css\_schemes\Gemini\index.styl

    // 文章内链接文本样式
.post-body p a{
  color: #0593d3; //原始链接颜色
  border-bottom: none;
  &:hover {
    color: #fc6423; //鼠标经过颜色
    border-bottom: 1px solid #fc6423;
  }
}

  1. 首页文章显示概览。在Front Matter中添加description。如下图。

    同时我想设置主页的文章在没有description和自动截取的excerpt的情况下，不显示文章具体内容。

    打开目录

    ..\themes\next\layout\_macro

    将post.swig文件复制一份，重命名为post-new.swig之类的。防止删掉不该删的东西（。）

    打开post-new.swig，control+F搜索if is_index，删除下面红框内的内容。我个人习惯是直接点标题进文章的，所以上面的阅读全文按钮也删了。

    如果觉得有点空想要加上标签，可以复制以下内容进行粘贴。（或者直接在footer里复制这段也行，就在往下面拉50行左右的地方）

    自己理一下几个if、elif和endif的配对关系，不要放错了。

    {%- if post.tags and post.tags.length %}
      {%- if theme.tag_icon %}
          {%- set tag_indicate = '<i class="fa fa-tag"></i>' %}
      {% else %} 
          {%- set tag_indicate = '#' %}
      {%- endif %}

      <div class="post-tags">
          {%- for tag in post.tags.toArray() %}
              <a href="{{ url_for(tag.path) }}" rel="tag">{{ tag_indicate }} {{ tag.name }}</a>
          {%- endfor %}
      </div>

  {%- endif %}

    自己理一下几个if、elif和endif的配对关系，不要放错了。

    现在我们回到目录

    ..\themes\next\layout\index.swig

    将里面的引用的post.swig改写成我们刚刚的post-new.swig。
